# Design Brief Template

Use this template for a decision-ready CCFA research-task brief. Do not write a file unless the user explicitly asks for a reusable brief, spec, or saved document.

## Compact Brief

```text
Decision:
Goal:
Audience:
Inputs:
Constraints:
Success criteria:
Recommended approach:
Next CCFA skill:
Handoff condition:
Open risks:
```

## Standard Brief

```text
Title:
Mode:

Decision To Make:

Goal And Audience:

Available Inputs:

Constraints And Privacy Boundary:

Success Criteria:

Scope Diagnosis:

Approach Options:

Recommended Route:

Next CCFA Skill:

Handoff Conditions:

Questions That Still Matter:

What Not To Do Yet:
```

## What Not To Do Yet

Use this section to prevent premature downstream work:

- do not optimize the idea until the target contribution type is clear;
- do not search with private wording unless authorized;
- do not design experiments before the central paper conclusion is stable;
- do not write manuscript prose before paper conclusions and evidence are aligned;
- do not compress before the user decides appendix/delete policy;
- do not draft rebuttal unless real reviewer comments or explicit rebuttal intent exist.

## File Output

If the user explicitly asks for a saved brief, use a short filename such as:

```text
ccfa-brief-<topic>.md
```

Keep the file in the user's requested location. If no location is given, ask once before writing because reusable file generation is a CCFA handoff-sensitive action.
